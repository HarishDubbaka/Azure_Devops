# 🌿 Branch Management with Git in Azure DevOps

## 📌 Overview

This exercise walks through **branch management** using:

* Visual Studio Code
* Azure DevOps Portal

Branching helps isolate work, making it easier to manage **features, bug fixes, and collaboration** without affecting the main codebase.

---

## 🧠 Key Concepts

* 🌱 **Branches** allow independent development without impacting other work
* 🔄 Changes in one branch do **not affect** others until merged
* 🚀 Lightweight structure makes switching branches fast
* 👥 Teams can collaborate by sharing branches
* 🔀 Pull Requests enable code review and controlled merging

---

## ⚙️ Why Use Branches?

✔️ Isolate new features or bug fixes
✔️ Avoid breaking the main branch
✔️ Enable parallel development
✔️ Simplify testing and code reviews
✔️ Improve team collaboration

---

## 🧪 Task 1: Create a New Branch (VS Code)

Follow these steps to create a branch in your local repository:

### 🔹 Step-by-Step Instructions

1. Open **Visual Studio Code**

2. Go to the **Source Control** tab

3. In the **bottom-left corner**, click on the current branch name (`main`)

4. From the popup, select:

   ➕ **Create new branch from...**

5. Choose `main` as the reference branch

6. Enter the branch name:

   ```bash
   dev
   ```

7. Press **Enter**

---

## 🔔 Note

* After creation, you will be **automatically switched** to the `dev` branch
* Any commits you make now will only affect this branch

---

## 🔄 Next Steps

* ✏️ Make changes and commit to `dev`
* 📤 Push branch to Azure DevOps
* 🔀 Create a Pull Request to merge into `main`
* 👀 Review code before merging
